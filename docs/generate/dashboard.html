azureml-examples files referenced in machine-learning docs

These dashboards show the status of all files from azureml-examples that are used in our machine learning documentation.

Use the links above for dashboards of each filetype. In the dashboards, click on the badge or text in the Status column to see the workflow runs. Click on the file link to open the associated file. (Or right click to open in a new tab.)

What do the badges mean?

You'll see these status badges in the dashboards:

Status        Description
Green passing The last workflow run on the main branch was successful. But it still could have been a long time ago. Select the badge to see the history of runs.
Grey no status I think this has something to do an absence of runs on main, even though there may be runs on one or more branches. Treat is the same as failing. Select the badge to see the history.
Red failing The last workflow run failed. Look at the workflow to determine how long it's been failing. Investigate when the failures are consistent. A single failure might be due to limited resources or other non-code related issues.
No badge There is no associated workflow for this file. (Or, there's something strange about the name and the way I construct the link failed. I need to investigate these more.)
